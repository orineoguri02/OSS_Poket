# Lapras 모델 파일 설정 가이드

이미지에서 보이는 파일들을 `public` 폴더에 다음과 같이 배치하세요:

## 파일 구조

```
my-3d-app/
└── public/
    ├── a131.dae                    ← DAE 모델 파일 (또는 a131.fbx)
    ├── Body.png                    ← 몸체 텍스처
    ├── Body_norm.png               ← 몸체 노말맵
    ├── Body_pow.png                ← 몸체 파워맵
    ├── Body_spc.png                ← 몸체 스펙큘러맵
    ├── Eyes.png                    ← 눈 텍스처
    ├── Eyes_norm.png               ← 눈 노말맵
    ├── Eyes_pow.png                ← 눈 파워맵
    ├── Eyes_spc.png                ← 눈 스펙큘러맵
    └── ...
```

## 설정 단계

1. **모델 파일 복사**
   - `a131.dae` 파일을 `public` 폴더에 복사합니다.
   - 또는 `a131.fbx` 파일을 사용하려면 `Detail.jsx`에서 경로를 `/a131.fbx`로 변경하세요.

2. **텍스처 파일 복사**
   - 모든 텍스처 파일들(`Body.png`, `Eyes.png` 등)을 `public` 폴더에 복사합니다.
   - 파일 이름을 그대로 유지하세요 (대소문자 구분).

3. **파일 확인**
   - `public` 폴더에 다음 파일들이 있는지 확인:
     - ✅ `a131.dae` (또는 `a131.fbx`)
     - ✅ `Body.png`
     - ✅ `Body_norm.png`
     - ✅ `Body_pow.png`
     - ✅ `Body_spc.png`
     - ✅ `Eyes.png`
     - ✅ `Eyes_norm.png`
     - ✅ `Eyes_pow.png`
     - ✅ `Eyes_spc.png`

4. **브라우저에서 확인**
   - 개발 서버를 실행하고 `/pokemon/131` 경로로 이동합니다.
   - Lapras 3D 모델이 텍스처와 함께 표시됩니다!

## 문제 해결

- **텍스처가 보이지 않으면:**
  1. 브라우저 콘솔(F12)에서 에러 메시지를 확인하세요.
  2. 텍스처 파일 이름이 정확한지 확인하세요 (대소문자 포함).
  3. DAE 파일 내부의 텍스처 경로가 상대 경로로 되어 있을 수 있습니다. 코드가 자동으로 절대 경로로 변환합니다.

- **모델이 보이지 않으면:**
  1. 파일 경로가 정확한지 확인하세요 (`/a131.dae`).
  2. 브라우저 콘솔에서 에러 메시지를 확인하세요.
  3. 파일이 `public` 폴더에 올바르게 저장되었는지 확인하세요.

- **모델 크기가 이상하면:**
  - `Lapras3D.jsx`의 `scale` 값을 조정하세요.
  - 현재 DAE 파일은 `scale={[2, 2, 2]}`로 설정되어 있습니다.
